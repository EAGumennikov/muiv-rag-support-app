# MUIV RAG Support App

Прототип системы управления знаниями службы поддержки пользователей с применением RAG-подхода.

## Что реализовано
- нормализация HTML-корпуса в Markdown
- чанкинг корпуса
- построение эмбеддингов
- построение FAISS-индекса
- извлечение retrieval-слоя по пользовательскому запросу
- построение ответа
- интеграция с Yandex AI Studio
- минимальный веб-интерфейс на Flask
- маршрут `/ask` с подключением к реальному RAG-контуру
- публичная структура сайта с 10+ страницами, хлебными крошками и каталогом статей
- просмотр статьи и скачивание markdown по `doc_id`
- SQL-слой приложения на SQLite с 10+ таблицами
- хранение обратной связи и истории запросов в SQL-базе

## Структура
- app.py - точка входа Flask-приложения
- db/ - базовый SQL-слой, конфигурация и модели данных
- services/ - сервисный слой retrieval и генерации ответа
- scripts/ - основные скрипты проекта
- templates/ - HTML-шаблоны веб-интерфейса
- static/ - статические ресурсы веб-интерфейса
- config/ - шаблоны конфигурации
- docs/ - технические заметки по проекту

## Контуры работы

### Репозиторий
Репозиторий содержит только код проекта, шаблоны конфигурации, веб-интерфейс и техническую документацию.

### Локальный рабочий каталог
Runtime-данные, виртуальное окружение, секреты и крупные артефакты хранятся вне репозитория в отдельном рабочем каталоге.

### Synology
Исходные данные, заметки, черновики и эксплуатационные артефакты проекта хранятся на Synology. В целевом сценарии запуск решения предполагается выполнять в данном контуре.

## Важно
Тяжелые данные, секреты, индексы и runtime-артефакты в репозиторий не включаются.

## Веб-интерфейс

На текущем этапе веб-слой включает:
- публичные страницы `/`, `/search`, `/articles`, `/glossary`, `/onboarding`, `/faq`, `/about-system`, `/about-kb`, `/testing`, `/feedback`
- хлебные крошки на всех пользовательских страницах
- маршрут `/ask` для обработки вопроса через реальный RAG-контур
- просмотр статьи `/article/<doc_id>`
- скачивание markdown `/download/<doc_id>`
- форму обратной связи

Веб-интерфейс использует существующий retrieval-контур и генерацию ответа через Yandex AI Studio по OpenAI-совместимому API.

## SQL-слой и SQLite

В приложении добавлен демонстрационный SQL-слой на базе `SQLAlchemy`.

Что уже хранится в базе:
- сообщения формы обратной связи;
- история поисковых запросов;
- итоговые RAG-ответы;
- связанные источники ответа;
- базовый аудит событий.

По умолчанию используется SQLite во внешнем `workdir`:

`sqlite:///~/Projects/muiv-rag-support-workdir/output/app/muiv_rag_support.sqlite3`

Путь можно переопределить через переменную окружения `RAG_APP_DATABASE_URL`.

Для явной инициализации таблиц можно выполнить:

```bash
cd ~/Projects/muiv-rag-support-app
source .venv/bin/activate
python3 scripts/init_app_db.py
```

## Локальный запуск веб-приложения

Пример запуска в существующей архитектуре `repo + workdir`:

```bash
cd ~/Projects/muiv-rag-support-app
source .venv/bin/activate
pip install -r requirements.txt
python3 app.py
```

После запуска интерфейс будет доступен по адресу [http://127.0.0.1:5000](http://127.0.0.1:5000).

Если нужно сразу поднять приложение с подключением переменных окружения из workdir, можно использовать отдельный скрипт:

```bash
cd ~/Projects/muiv-rag-support-app
bash scripts/run_web.sh
```

## Соотношение repo и workdir

- `~/Projects/muiv-rag-support-app` - код, шаблоны, Flask-приложение, CLI-скрипты, документация
- `~/Projects/muiv-rag-support-workdir` - `.env`, индексы, embeddings, выходные артефакты и прочие runtime-данные

Такое разделение сохраняется и для веб-слоя: код интерфейса остается в репозитории, а рабочие данные по-прежнему должны храниться во внешнем workdir.
