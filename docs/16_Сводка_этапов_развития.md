# Сводка этапов развития проекта

Документ содержит краткую публикуемую сводку развития проекта без включения секретов, runtime-данных, внутренних черновиков отчета и чувствительной информации.

| 2 | Подготовлен чанкинг корпуса для semantic retrieval. |
| 3 | Проект перенесен в разделенную структуру кода и рабочего каталога. |
| 4 | Выполнен переход от retrieval-слоя к формированию ответа. |
| 5 | Собран answer pipeline для ответа по найденному контексту. |
| 6 | Изучены параметры подключения YandexGPT API и внешнего LLM-контура. |
| 7 | Подключена генеративная модель и выполнена первичная оценка стоимости запроса. |
| 8 | Оптимизированы объем контекста, prompt и стоимость генерации ответа. |
| 9 | Закреплена архитектура `repo + workdir` для кода и runtime-данных. |
| 10 | Добавлены локальные скрипты запуска. |
| 11 | Уточнены параметры генерации ответа. |
| 12 | Flask-приложение подключено к реальному RAG-контуру. |
| 13 | Расширен публичный контур web-сервиса. |
| 14 | Улучшено отображение ответа и источников в интерфейсе. |
| 15 | Спроектирован SQL-слой web-сервиса. |

| 18 | Реализованы роли, вход и базовый закрытый контур. |

| 26 | Актуализирована структура публичных разделов сайта. |
| 27 | Уточнены целевые аудитории и сценарии использования. |
| 28 | Подготовлены Docker-контур, DS920+ и публикация через reverse proxy. |
| 29 | Переработан каталог базы знаний и улучшена доступность интерфейса. |
| 30 | Раздел поиска переименован в «RAG-поиск» без изменения URL `/search`. |
| 31 | README и репозиторная документация приведены к итоговому состоянию проекта. |

## Итог

К концу этапа 31 проект представляет собой опубликованный демонстрационный web-сервис с RAG-поиском, базой знаний, ролями, SQL-хранением, экспортом документов, справочными разделами и актуальной репозиторной документацией.
