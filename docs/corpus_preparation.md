# Подготовка корпуса знаний

Корпус знаний подготавливается из публичных HTML-материалов и приводится к формату, пригодному для поиска, индексации и отображения в web-интерфейсе.

## Исходный материал

В качестве основного массива использовалась публичная база знаний Astra Linux. Исходные материалы были представлены HTML-страницами с вложениями, изображениями и служебной разметкой.

Зафиксированные показатели подготовки:

- исходных HTML-файлов: 13118;
- успешно нормализовано в Markdown: 13105;
- ошибок нормализации: 13.

Ошибки связаны с отсутствием извлекаемого текстового содержимого или нестандартной структурой отдельных страниц.

## Нормализация HTML в Markdown

Нормализация выполняет:

- извлечение заголовка;
- извлечение breadcrumbs;
- извлечение автора и даты при наличии;
- очистку служебной HTML-разметки;
- удаление дублей заголовков;
- сохранение основного текста;
- сохранение результата в Markdown с metadata.

Пользовательский интерфейс не должен показывать внутренние пути и служебные поля. Для пользователя важны название статьи, структура разделов и понятное содержание.

## Чанкинг

После нормализации документы разбиваются на фрагменты для retrieval.

Финальные параметры чанкинга:

```text
target_chars = 1300
overlap_chars = 150
min_chars = 180
merge_smaller_than = 0
```

Результат чанкинга:

- 45099 чанков;
- средний размер чанка около 867 символов;
- агрегированный JSONL используется как основной вход для индексации.

## Индексация

Для semantic retrieval используются:

- модель embeddings `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`;
- FAISS-индекс;
- карта чанков;
- статистика индексирования.

Индексы и embeddings являются runtime-артефактами и не включаются в репозиторий.

## Пользовательское представление статьи

Страница статьи строится поверх подготовленных metadata и текста:

- показывает заголовок;
- показывает breadcrumbs;
- показывает автора и дату при наличии;
- рендерит Markdown как HTML;
- позволяет скачать Markdown;
- позволяет экспортировать статью в `.docx`;
- не показывает `source_file` и внутренние пути.
