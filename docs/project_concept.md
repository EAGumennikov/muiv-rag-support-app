# Концепция проекта

Проект представляет собой учебный web-сервис управления знаниями службы поддержки пользователей с применением RAG-подхода для поиска и формирования решений.

Утвержденная тема проекта: «Совершенствование системы управления знаниями службы поддержки пользователей с применением RAG-подхода для поиска и формирования решений».

Полное название web-сервиса: «Система управления знаниями службы поддержки пользователей с применением RAG-подхода для поиска и формирования решений».

Короткое отображаемое название: «Система управления знаниями поддержки».

Проект выполнен при прохождении преддипломной практики и подготовке ВКР в Московском Университете имени С.Ю. Витте (МУИВ). Студент: Гуменников Евгений Андреевич.

## Назначение

Система предназначена для поддержки сотрудников, которые работают с базой знаний и типовыми обращениями пользователей. Веб-сервис объединяет:

- публичный информационный контур;
- каталог статей базы знаний;
- RAG-поиск по подготовленному корпусу;
- личный кабинет сотрудника поддержки;
- контур редактора знаний;
- административный контур;
- SQL-хранение прикладных данных;
- экспорт ответов, статей, истории и отчетов.

## Целевая идея

RAG-модуль не заменяет весь web-сервис, а является его прикладным поисковым ядром. Пользователь формулирует вопрос, система находит релевантные фрагменты базы знаний, формирует ответ и показывает источники, на которые он опирается.

Такой подход позволяет:

- ускорить поиск решений;
- повысить прозрачность ответа за счет источников;
- сохранять историю вопросов и ответов;
- анализировать качество базы знаний;
- использовать накопленные данные в административном контуре.

## Границы первой версии

В первой версии реализуются демонстрационные, но прикладные сценарии:

- подготовка и индексация корпуса;
- поиск и формирование ответа;
- публичные страницы web-ресурса;
- каталог статей;
- обратная связь;
- роли и кабинеты;
- административные страницы;
- экспорт документов.

В первую версию не входит полноценная система заявок, сложный workflow обработки инцидентов, автоматическое обновление корпуса из внешних источников и промышленный мониторинг качества LLM-ответов.

## Целевой демонстрационный контур

Для соответствия требованиям сетевой ресурс должен быть размещен на бесплатном хостинге. Локальная архитектура `repo + workdir` используется для разработки и проверки, а демонстрационное размещение должно быть описано отдельным этапом с адресом, ограничениями и порядком развертывания.

## Визуальная концепция

Интерфейс оформляется как сдержанный учебный ИТ-сервис: светлый фон, темный текст, синие акценты, тонкие границы, строгие карточки, единые формы, кнопки и таблицы. В шапке используется нейтральная декоративная пиктограмма атома, не связанная с официальными логотипами или внешними брендовыми материалами.
