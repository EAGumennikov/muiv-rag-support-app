# Размещение и запуск

Итоговый сетевой ресурс размещен в формате self-hosting на демонстрационном сервере DS920+. Такой вариант используется как бесплатный demo-контур для демонстрации web-сервиса вне локальной машины.

## Цель размещения

Размещение нужно для демонстрации работоспособного web-сервиса вне локальной машины. В отчете можно указать:

- адрес опубликованного ресурса;
- выбранный бесплатный хостинг;
- порядок развертывания;
- ограничения демонстрационной среды;
- доступность публичных страниц;
- демонстрационные учетные записи для проверки закрытых разделов.

## Что нужно учитывать

Приложение использует:

- Flask;
- SQLAlchemy;
- SQLite для локальной демонстрации;
- внешние runtime-артефакты RAG;
- переменные окружения для секретов и настроек.

При размещении нельзя публиковать:

- API-ключи;
- `.env`;
- embeddings;
- FAISS-индексы, если площадка не рассчитана на такие файлы;
- локальные пользовательские данные;
- временные выгрузки.

## Проверенные условия демонстрации

При эксплуатации demo-контура учитываются:

- возможность установки зависимостей из `requirements.txt`;
- способ запуска Flask-приложения;
- поддержку постоянного или временного файлового хранилища;
- ограничения по размеру файлов;
- ограничения по времени выполнения запроса;
- способ хранения SQLite-БД или необходимость перехода на внешнюю SQL-СУБД.

## Подготовка к развертыванию

Перед размещением были выполнены следующие подготовительные действия:

1. Зафиксирован минимальный набор runtime-данных во внешнем `workdir`.
2. Проверено, что секреты, индексы, embeddings и SQLite-БД не попадают в git и Docker-образ.
3. Секреты оставлены во внешних `.env`-файлах рабочего контура.
4. Старт приложения переведен на относительную схему `app + workdir`.
5. Подготовлены `Dockerfile`, `.dockerignore` и `compose.yaml`.
6. Проверены публичный контур, HTTPS-доступ и адаптивность опубликованных страниц.

## Защита RAG-контура перед публикацией

Перед размещением на бесплатном хостинге маршрут `/ask` защищен авторизацией. Публичные страницы, каталог статей, глоссарий, FAQ, сведения о проекте и форма обратной связи остаются открытыми, но запуск retrieval и внешнего генеративного API доступен только ролям `support_user`, `knowledge_editor` и `admin`.

Для demo-контура добавлен дневной лимит RAG-запросов. Он задается переменной окружения `RAG_DAILY_LIMIT`, по умолчанию равен `10` и считается по SQL-истории текущего пользователя. Лимит проверяется до обращения к retrieval и внешнему API. Для роли `admin` лимит не применяется, чтобы администратор мог проводить демонстрационную проверку.

## Статус размещения

Требование размещения закрыто: приложение опубликовано через Docker и обратный прокси Synology. Кодовый репозиторий отделен от runtime-контура: `.env`, индексы, embeddings, SQLite-БД, логи и временные выгрузки не попадают в git.

## Self-hosting на демонстрационном сервере DS920+

После подтверждения возможности использовать собственный NAS выбран вариант бесплатного размещения в формате self-hosting на демонстрационном сервере DS920+. Это позволяет показать работающий сетевой ресурс без передачи секретов, индексов, embeddings и SQLite-БД стороннему хостингу.

Принятая схема:

- код приложения хранится в `/volume1/MUIV_Diplom/04_development/muiv-rag-support-app`;
- runtime-данные RAG-контура хранятся отдельно в `/volume1/MUIV_Diplom/04_development/muiv-rag-support-workdir`;
- Docker-образ собирается из репозитория приложения;
- `workdir` подключается в контейнер как volume `/workdir`;
- `.env.local` и `.env.yc` остаются вне git и Docker-образа;
- контейнер `muiv-rag-support` слушает порт `5002`;
- локальная проверка выполняется по адресу `http://192.168.2.50:5002`;
- публичный доступ настроен через обратный прокси Synology и домен `https://muiv.eagumennikov.synology.me`.

Файлы Docker-контура:

- `Dockerfile` - сборка Python-образа, установка CPU-only PyTorch, зависимостей проекта и запуск через Gunicorn;
- `.dockerignore` - исключение `.git`, `.venv`, `.env`, SQLite-БД, логов, экспортов, индексов, embeddings и других runtime-артефактов из контекста сборки;
- `compose.yaml` - подключение `workdir`, настройка переменных окружения, host network и команды запуска.

Команда запуска на DS920+:

```bash
cd /volume1/MUIV_Diplom/04_development/muiv-rag-support-app
sudo -n /usr/local/bin/docker compose up -d --build
```

Проверка контейнера:

```bash
sudo -n /usr/local/bin/docker ps
sudo -n /usr/local/bin/docker logs --tail 80 muiv-rag-support
```

Особенности текущего контура:

- bridge-сеть Docker на DS920+ в текущей конфигурации не получает корректный DNS при сборке, поэтому в `compose.yaml` для сборки и запуска используется host network;
- внешний порт выбран `5002`, так как порт `5000` на NAS занят другим процессом;
- PyTorch устанавливается в CPU-only варианте `torch==2.4.1+cpu`, чтобы исключить CUDA-зависимости и уменьшить Docker-образ;
- текущий размер рабочего образа после оптимизации составляет около 1.55 GB;
- перенос исходных папок на NAS не является изменением приложения, поэтому фиксируется в проектных заметках, а не отдельным коммитом.

## Проверки на DS920+

После подготовки Docker-контура выполнены проверки:

- контейнер `muiv-rag-support` запущен через Docker Compose;
- публичные маршруты `/`, `/search`, `/articles`, `/glossary`, `/diagnostics`, `/onboarding`, `/testing`, `/faq`, `/about-project`, `/feedback`, `/login` возвращают HTTP 200 на `127.0.0.1:5002`;
- доступ из локальной сети по `http://192.168.2.50:5002` подтвержден для `/`, `/search`, `/about-project`;
- проверено, что runtime-данные остаются во внешнем `workdir`, а не в Docker-образе и не в git.

Обратный прокси Synology и HTTPS-доступ по доменному имени `https://muiv.eagumennikov.synology.me` настроены. Закрытые сценарии, экспорт и RAG-контур дополнительно проверяются через Flask test client и при демонстрации с авторизованными учетными записями.


## HTTPS и адаптивная проверка

Публичный адрес demo-контура: `https://muiv.eagumennikov.synology.me`.

Проверено:

- reverse proxy Synology направляет внешний HTTPS-адрес на `http://192.168.2.50:5002`;
- сертификат Let's Encrypt для `*.eagumennikov.synology.me` покрывает поддомен `muiv.eagumennikov.synology.me`;
- проверка TLS через `openssl s_client` завершилась с `Verify return code: 0 (ok)`;
- публичные маршруты `/`, `/search`, `/about-project`, `/login` возвращают HTTP 200 через HTTPS;
- адаптивность публичного контура проверена на desktop `1440x900`, tablet `768x1024` и mobile `390x844`;
- горизонтальное переполнение на ключевых публичных страницах не выявлено.
