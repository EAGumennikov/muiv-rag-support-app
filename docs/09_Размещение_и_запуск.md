# Размещение на бесплатном хостинге

Итоговый сетевой ресурс должен быть размещен на бесплатном хостинге. Это отдельный обязательный этап, который должен быть закрыт технически и документально.

## Цель размещения

Размещение нужно для демонстрации работоспособного web-сервиса вне локальной машины. В отчете необходимо указать:

- адрес опубликованного ресурса;
- выбранный бесплатный хостинг;
- порядок развертывания;
- ограничения демонстрационной среды;
- доступность публичных страниц;
- демонстрационные учетные записи для проверки закрытых разделов.

## Что нужно учитывать

Приложение использует:

- Flask;
- SQLAlchemy;
- SQLite для локальной демонстрации;
- внешние runtime-артефакты RAG;
- переменные окружения для секретов и настроек.

При размещении нельзя публиковать:

- API-ключи;
- `.env`;
- embeddings;
- FAISS-индексы, если площадка не рассчитана на такие файлы;
- локальные пользовательские данные;
- временные выгрузки.

## Возможные варианты демонстрации

Для бесплатного хостинга можно рассмотреть платформу, поддерживающую Python web-приложения и переменные окружения. На этапе размещения нужно отдельно проверить:

- возможность установки зависимостей из `requirements.txt`;
- способ запуска Flask-приложения;
- поддержку постоянного или временного файлового хранилища;
- ограничения по размеру файлов;
- ограничения по времени выполнения запроса;
- способ хранения SQLite-БД или необходимость перехода на внешнюю SQL-СУБД.

## Подготовка к развертыванию

Перед размещением рекомендуется:

1. Зафиксировать минимальный набор runtime-данных.
2. Проверить, какие файлы корпуса и индекса допустимо использовать на выбранной площадке.
3. Перенести секреты в переменные окружения.
4. Проверить старт приложения без локальных абсолютных путей.
5. Подготовить инструкцию запуска.
6. Проверить публичный контур, роли, feedback, экспорт и RAG-сценарий.

## Защита RAG-контура перед публикацией

Перед размещением на бесплатном хостинге маршрут `/ask` защищен авторизацией. Публичные страницы, каталог статей, глоссарий, FAQ, сведения о системе и форма обратной связи остаются открытыми, но запуск retrieval и внешнего генеративного API доступен только ролям `support_user`, `knowledge_editor` и `admin`.

Для demo-контура добавлен дневной лимит RAG-запросов. Он задается переменной окружения `RAG_DAILY_LIMIT`, по умолчанию равен `10` и считается по SQL-истории текущего пользователя. Лимит проверяется до обращения к retrieval и внешнему API. Для роли `admin` лимит не применяется, чтобы администратор мог проводить демонстрационную проверку.

## Ограничения

Бесплатный хостинг может ограничивать объем диска, время выполнения процесса и доступность фоновых задач. Эти ограничения необходимо явно описать в отчете и учитывать при демонстрации RAG-контура.
