# Проверка локального RAG-контура

На данном этапе выполнено разделение проекта на два контура:
- кодовый репозиторий `muiv-rag-support-app`, содержащий только программный код и служебную документацию;
- рабочий каталог `muiv-rag-support-workdir`, содержащий входные данные, выходные артефакты, индексы, эмбеддинги и локальные конфигурационные файлы запуска.

В основных скриптах проекта убраны жестко заданные пути к прежней локальной структуре `MUIV_Diplom_local`. Для этого добавлен общий модуль конфигурации путей, через который задаются пути к:
- входному JSONL-корпусу чанков;
- файлу эмбеддингов;
- FAISS-индексу;
- chunk map;
- файлу статистики индексирования.

После внесения изменений выполнен повторный запуск этапов индексирования и генерации ответа уже в новой структуре. В результате:
- корректно созданы выходные файлы в каталоге `output`;
- semantic retrieval успешно извлекает релевантные фрагменты;
- answer pipeline формирует итоговый ответ на основе retrieval-контекста и внешней LLM.

Тестовый запрос по настройке приложения по умолчанию для определенного типа файлов был успешно обработан, что подтвердило работоспособность связки:
корпус чанков -> эмбеддинги -> индекс -> retrieval -> генерация ответа.

Таким образом, на данном этапе подтверждена работоспособность локального прототипа RAG-контура в разделенной архитектуре `repo + workdir`. Целевой демонстрационный запуск web-сервиса должен быть оформлен отдельно на бесплатном хостинге.
