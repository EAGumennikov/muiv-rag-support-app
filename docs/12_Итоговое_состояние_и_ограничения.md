# Итоговое состояние и ограничения

Документ фиксирует состояние web-сервиса после реализации основного функционального контура, публикации на демонстрационном сервере DS920+ и актуализации репозиторной документации.

## Итоговое состояние

Проект готов к использованию как демонстрационный сетевой ресурс для отчета по преддипломной практике:

- публичный контур содержит 10 основных разделов и страницы статей;
- RAG-поиск работает через защищенный маршрут `/ask`;
- каталог базы знаний поддерживает поиск, фильтрацию, сортировку и пагинацию;
- SQL-слой хранит пользователей, роли, историю запросов, RAG-ответы, источники, feedback и аудит;
- реализованы роли `support_user`, `knowledge_editor` и `admin`;
- личный кабинет, редакторский контур и административная панель закрыты ролевыми проверками;
- экспорт `.docx` и `.xlsx` формируется в памяти без сохранения пользовательских файлов в репозиторий;
- глоссарий и диагностический справочник поддерживают сценарий адаптации сотрудников;
- опубликованный demo-контур работает на DS920+ через Docker, Gunicorn, reverse proxy и HTTPS.

## Границы текущей версии

Текущая версия является учебным демонстрационным web-сервисом, а не промышленной системой управления заявками. В нее намеренно не включены:

- полноценный workflow обработки инцидентов;
- автоматическое обновление корпуса из внешних источников;
- промышленный мониторинг качества LLM-ответов;
- сложная ролевая матрица с подразделениями и группами доступа;
- хранение секретов, индексов, embeddings и runtime-БД в репозитории;
- публикация черновиков или готовых материалов отчета по практике и ВКР.

## Зафиксированные ограничения

- Глоссарий и диагностический справочник хранятся в сервисном слое, потому что текущая SQL-модель `glossary_terms` не содержит всех полей, нужных интерфейсу.
- CRUD для справочников в редакторском контуре не реализован, так как для текущей версии достаточно демонстрационного наполнения.
- Административные таблицы не имеют расширенных фильтров по датам и статусам, поскольку объем демонстрационных данных ограничен.
- Экспортные документы используют базовое оформление без настраиваемых шаблонов.
- SQLite-БД используется как демонстрационная SQL-СУБД и хранится во внешнем runtime-контуре.

## Направления возможного развития

После подготовки отчета по практике проект можно развивать в следующих направлениях:

- перенести справочники в управляемый SQL-контур с CRUD для редактора знаний;
- подготовить ER-схему БД для включения в отчетные материалы;
- добавить фильтры административных выгрузок по датам, статусам и ролям;
- расширить аналитику качества источников и обращений feedback;
- добавить регламент обновления корпуса знаний и пересборки индекса;
- подготовить отдельный набор демонстрационных данных для публичного показа.

Эти направления не блокируют текущую готовность проекта к отчету по практике, но полезны как основа для последующего развития ВКР.
