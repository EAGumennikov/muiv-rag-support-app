# Веб-интерфейс и пользовательские сценарии

Web-интерфейс реализован на Flask и Jinja-шаблонах. Он включает публичный контур, личный кабинет, редакторский контур и административные страницы.

## Публичный контур

Публичные страницы доступны без входа:

- `/` - главная страница;
- `/search` - поиск по базе знаний;
- `/articles` - каталог статей;
- `/article/<doc_id>` - страница статьи;
- `/download/<doc_id>` - скачивание Markdown;
- `/article/<doc_id>/export.docx` - экспорт статьи в `.docx`;
- `/glossary` - глоссарий;
- `/onboarding` - сценарий первого использования;
- `/faq` - часто задаваемые вопросы;
- `/about-system` - описание системы;
- `/about-kb` - описание базы знаний;
- `/testing` - подходы к тестированию;
- `/feedback` - форма обратной связи;
- `/login` - вход.

## Сценарий RAG-поиска

Пользователь открывает `/search`. Страница остается публичной, но формирование ответа относится к защищенному контуру и доступно только авторизованным пользователям с ролями `support_user`, `knowledge_editor` или `admin`.

Для неавторизованного пользователя вместо активной формы показывается информационный блок с предложением войти. Это не скрывает публичный раздел, но не позволяет случайно запустить retrieval и внешний генеративный API.

Для авторизованного пользователя форма отправляет вопрос на `/ask` через AJAX / fetch без перезагрузки страницы. Интерфейс:

- блокирует кнопку на время обработки;
- показывает индикатор формирования ответа;
- выводит текст ответа на текущей странице;
- обновляет карточки источников;
- показывает ссылку на экспорт RAG-ответа в `.docx`, если ответ сохранен в SQL;
- выводит понятные ошибки авторизации, дневного лимита, пустого вопроса и сбоя обработки.

Если JavaScript отключен, HTML-форма продолжает работать через обычный POST-запрос на `/ask`. После обработки пользователь видит:

- исходный вопрос;
- ответ;
- карточки источников;
- ссылки на статьи;
- возможность скачать Markdown-материал статьи;
- возможность скачать сформированный ответ в `.docx`, если он сохранен.

Если пользователь авторизован, вопрос и ответ сохраняются с привязкой к учетной записи.

## Сценарий статьи

Каталог `/articles` показывает список документов корпуса. Страница статьи содержит:

- заголовок;
- breadcrumbs;
- текст материала;
- сведения о статье;
- скачивание Markdown;
- экспорт `.docx`;
- ссылку на оригинал при наличии.

## Сценарий обратной связи

Форма `/feedback` сохраняет обращение в SQL-БД. Если пользователь авторизован, обращение связывается с его учетной записью. Администратор видит обращения в закрытом контуре и может менять статус.

## Закрытые контуры

Закрытые страницы используют Flask-сессию и ролевые проверки:

- личный кабинет доступен авторизованным пользователям;
- редакторский контур доступен `knowledge_editor` и `admin`;
- административный контур доступен только `admin`.

## Навигация

Навигация разделена на:

- основное публичное меню;
- навигацию личного кабинета;
- навигацию редакторского контура;
- административную навигацию.

На пользовательских страницах используются хлебные крошки.
