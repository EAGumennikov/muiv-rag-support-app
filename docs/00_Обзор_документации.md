# Документация проекта

Раздел содержит репозиторные статьи по архитектуре, данным, RAG-контуру, web-интерфейсу, размещению, проверкам и требованиям проекта. Документы предназначены для сопровождения кода и подготовки отчета по практике и ВКР.

На этапе 24 документы переименованы на русском языке с числовой сортировкой. Это упрощает навигацию по материалам и позволяет использовать `docs/` как основу для проектной части отчета без изменения технических имен модулей приложения.

## Ограничения репозиторной документации

В репозиторий не включаются секреты, API-ключи, `.env`, embeddings, индексы, runtime-файлы, пользовательские выгрузки и локальные рабочие данные. Документация описывает архитектуру и сценарии работы без публикации чувствительных параметров окружения.

## Ревизия текущего состояния

Сводная проверка требований, маршрутная карта Flask-приложения, инвентаризация SQL-таблиц, подсчет логических строк кода и границы между репозиторием и runtime-контуром зафиксированы в [трассировке требований](10_Трассировка_требований.md). Итоговые проверки опубликованного контура и локальных сценариев собраны в документе [Проверки и приемка текущего состояния](15_Проверки_и_приемка.md).

Служебный подсчет объема кода выполняется командой:

```bash
python3 scripts/count_logical_lines.py
```

На момент ревизии результат составляет `6368` логических строк без учета документации, виртуального окружения, `.git` и runtime-артефактов.
