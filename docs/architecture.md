# Архитектура приложения

Приложение построено как Flask-based web-сервис с разделением на маршруты, сервисный слой, SQL-модели, шаблоны и статические ресурсы.

## Основные контуры

- `app.py` - точка входа Flask-приложения, маршруты и связывание web-уровня с сервисами.
- `services/` - прикладная логика: RAG, статьи, SQL-операции, доступ, markdown и экспорт.
- `db/` - SQLAlchemy Base, конфигурация подключения и ORM-модели.
- `templates/` - HTML-шаблоны публичных и закрытых страниц.
- `static/` - стили web-интерфейса.
- `scripts/` - CLI-сценарии подготовки корпуса, индексации, запуска и служебной инициализации.
- `docs/` - репозиторная документация по архитектуре и требованиям.

## Разделение repo и workdir

Кодовый репозиторий содержит только исходный код, шаблоны, стили, конфигурационные примеры, зависимости и документацию.

Внешний рабочий каталог используется для runtime-данных:

- `.env`;
- FAISS-индекс;
- embeddings;
- агрегированные чанки;
- выходные артефакты;
- локальная SQLite-БД;
- временные файлы запуска.

Такое разделение снижает риск попадания секретов, тяжелых файлов и локальных данных в git.

## Сервисный слой

Сервисный слой отделяет бизнес-логику от Flask-маршрутов:

- `answer_service.py` формирует ответ на основе retrieval-контекста;
- `retrieval_service.py` отвечает за поиск релевантных фрагментов;
- `article_service.py` собирает пользовательское представление статей;
- `db_service.py` скрывает SQLAlchemy-операции;
- `auth_service.py` реализует аутентификацию и роли;
- `access_content_service.py` описывает закрытые разделы;
- `site_content_service.py` хранит публичные тексты;
- `export_service.py` формирует `.docx` и `.xlsx` в памяти.

## Web-уровень

Flask-приложение включает:

- публичные маршруты;
- маршрут `/ask` для RAG-запроса;
- страницу статьи и скачивание Markdown;
- экспорт статьи в `.docx`;
- вход и выход;
- личный кабинет;
- редакторский контур;
- административный контур;
- экспортные маршруты.

## Принцип безопасности данных

В репозиторий не добавляются:

- API-ключи;
- `.env`;
- embeddings;
- индексы;
- runtime-логи;
- пользовательские выгрузки;
- локальные рабочие данные.

Все чувствительные параметры должны передаваться через переменные окружения или внешний рабочий контур.
