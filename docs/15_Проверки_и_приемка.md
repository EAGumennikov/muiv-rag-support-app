# Проверки и приемка текущего состояния

Документ фиксирует проверки, выполненные для текущей версии web-сервиса после публикации demo-контура и актуализации интерфейса.

## Проверки опубликованного контура

Публичный demo-контур доступен по адресу `https://muiv.eagumennikov.synology.me`.

Проверено:

- главная страница открывается через HTTPS;
- публичные разделы `/search`, `/articles`, `/glossary`, `/diagnostics`, `/onboarding`, `/testing`, `/faq`, `/about-project`, `/feedback`, `/login` возвращают HTTP 200;
- reverse proxy Synology направляет внешний HTTPS-адрес на внутренний сервис DS920+;
- сертификат Let's Encrypt покрывает поддомен `muiv.eagumennikov.synology.me`;
- проверка TLS через `openssl s_client` завершилась с `Verify return code: 0 (ok)`.

## Проверка адаптивности

Публичный контур проверен в браузере на трех размерах viewport:

- desktop `1440x900`;
- tablet `768x1024`;
- mobile `390x844`.

Проверялись главная страница, RAG-поиск, база знаний, глоссарий, диагностика, раздел «Новичку», проверка знаний, FAQ, «О проекте» и обратная связь. Горизонтальное переполнение на ключевых публичных страницах не выявлено.

## Проверки через Flask test client

Для прикладных сценариев использовался Flask test client с тестовой SQLite-БД. Проверялись:

- публичные маршруты;
- совместимые redirect-маршруты `/about-system`, `/about-kb` и `/about`;
- каталог `/articles`, поиск по всему корпусу, пустой результат, фильтры и пагинация;
- просмотр статьи `/article/<doc_id>`;
- скачивание Markdown `/download/<doc_id>`;
- HTML- и JSON-ветки `/ask`;
- отказ неавторизованному пользователю до retrieval и внешнего API;
- доступ к `/ask` для ролей `support_user`, `knowledge_editor` и `admin`;
- срабатывание дневного лимита RAG-запросов до запуска retrieval;
- личный кабинет, редакторский контур и административные страницы;
- запрет закрытых страниц для неавторизованного пользователя;
- разграничение доступа между `support_user`, `knowledge_editor` и `admin`.

## Проверки экспорта

Проверены сценарии формирования файлов в памяти:

- экспорт RAG-ответа в `.docx`;
- экспорт статьи базы знаний в `.docx`;
- экспорт пользовательской истории в `.xlsx`;
- административная выгрузка feedback в `.xlsx`;
- административная выгрузка общей истории запросов в `.xlsx`;
- административная выгрузка статистики в `.xlsx`.

Пользовательские выгрузки не сохраняются в репозитории и не добавляются в git.

## Проверки кода и JavaScript

Для этапов интерфейса и каталога базы знаний выполнялись:

```bash
python -m compileall app.py db services scripts
node --check static/js/search.js
node --check static/js/reference_filters.js
```

Для подтверждения объема кода используется:

```bash
python3 scripts/count_logical_lines.py
```

На момент ревизии зафиксировано `6368` логических строк по `app.py`, `db/`, `services/`, `scripts/`, `templates/`, `static/css/` и `static/js/` без учета документации, `.git`, `.venv` и runtime-артефактов.

Основные требования к учебному сетевому ресурсу закрыты: Python + Flask, git-репозиторий, публичные страницы, SQL-СУБД, роли, закрытые контуры, RAG-поиск, экспорт документов, адаптивная верстка и размещение на опубликованном demo-контуре.
