# SQL-слой приложения

Документ фиксирует текущее состояние демонстрационного SQL-слоя веб-приложения.

## Технологическое решение

- используется `SQLAlchemy` без внешней серверной инфраструктуры;
- базовая СУБД для локального запуска — `SQLite`;
- путь к БД настраивается через `RAG_APP_DATABASE_URL`;
- по умолчанию база размещается во внешнем `workdir`, а не внутри репозитория.

## Базовый путь по умолчанию

Если переменная окружения `RAG_APP_DATABASE_URL` не задана, приложение использует:

`sqlite:///<workdir>/output/app/muiv_rag_support.sqlite3`

## Подготовленные таблицы

- `users`
- `roles`
- `user_roles`
- `articles`
- `article_files`
- `glossary_terms`
- `onboarding_pages`
- `faq_items`
- `feedback_messages`
- `search_queries`
- `rag_answers`
- `rag_answer_sources`
- `audit_logs`
- `settings`

## Что уже переведено на SQL

- хранение сообщений формы обратной связи;
- сохранение истории пользовательских запросов;
- сохранение итоговых RAG-ответов;
- сохранение связанных источников ответа;
- аудит базовых событий приложения;
- хранение пользователей, ролей и связей доступа.
- персональная привязка истории запросов и feedback к пользователю при наличии авторизации.

## Контур доступа

В текущей версии приложения реализован рабочий слой авторизации и ролевого доступа:

- `support_user`
- `knowledge_editor`
- `admin`

Поддерживаются:
- сессионный вход и выход через Flask;
- проверка авторизации на маршрутах;
- проверка ролей через helper/decorator-функции;
- демонстрационная инициализация тестовых учетных записей;
- персонализированная история вопросов, ответов и feedback;
- административный просмотр общей истории и обращений.

## Что будет развиваться дальше

- фильтры и расширенная аналитика по SQL-данным;
- схема БД для отчета;
- дополнительные административные сценарии работы с данными;
- расширение экспортов и отчетов;
- при необходимости переход на другую SQL-СУБД без смены прикладного слоя.
