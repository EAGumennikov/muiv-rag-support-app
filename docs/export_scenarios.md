# Экспорт документов и файловые сценарии

В проекте реализованы экспортные сценарии `.docx` и `.xlsx`. Они нужны для пользовательских выгрузок, административных отчетов и выполнения требования к формированию документов.

## Технологии

Для формирования документов используются:

- `python-docx` для `.docx`;
- `openpyxl` для `.xlsx`;
- `BytesIO` для формирования файла в памяти.

Экспортные файлы не сохраняются в репозиторий и не должны попадать в git.

## DOCX: RAG-ответ

Маршрут:

```text
/export/rag-answer/<rag_answer_id>.docx
```

Документ содержит:

- заголовок;
- дату формирования;
- дату исходного запроса;
- пользователя при наличии;
- текст вопроса;
- текст ответа;
- список источников;
- breadcrumbs и ссылку на оригинал при наличии.

Служебный prompt и внутренние технические поля не выводятся.

Доступ:

- `support_user` выгружает только свои ответы;
- `knowledge_editor` может выгружать ответы для анализа источников;
- `admin` может выгружать любые ответы.

## DOCX: статья базы знаний

Маршрут:

```text
/article/<doc_id>/export.docx
```

Документ содержит:

- название статьи;
- breadcrumbs;
- автора и дату при наличии;
- основной текст статьи;
- ссылку на оригинал при наличии;
- дату формирования.

Экспорт публичный, поскольку сама статья уже доступна через публичный маршрут.

## XLSX: моя история

Маршрут:

```text
/cabinet/history/export.xlsx
```

Файл содержит:

- дату запроса;
- вопрос;
- количество найденных источников;
- количество использованных источников;
- признак наличия ответа;
- краткий фрагмент ответа.

Выгрузка всегда строится только по текущему пользователю.

## XLSX: административные отчеты

Административные маршруты:

```text
/admin/export/feedback.xlsx
/admin/export/history.xlsx
/admin/export/statistics.xlsx
```

Выгрузка feedback содержит обращения пользователей и их статусы.

Выгрузка общей истории содержит все запросы, включая публичные запросы без пользователя.

Выгрузка статистики содержит:

- количество пользователей;
- количество ролей;
- количество feedback;
- количество поисковых запросов;
- количество RAG-ответов;
- распределение feedback по статусам;
- последние события аудита.

Все административные выгрузки доступны только роли `admin`.
